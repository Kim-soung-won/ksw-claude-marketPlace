# test-reviewer 리포트 출력 형식

감사 결과를 반환할 때 아래 형식을 그대로 사용한다. 파일을 수정하지 않으므로 리포트만
출력하고, 수정이 필요한 항목은 test-writer로 위임하도록 안내한다.

---

```
## 🔍 테스트 신뢰도 감사 결과

**대상**: {검토한 파일/디렉토리 — Focused | test/ 전체 — Full Scan}
**판정**: {PASS / 개선 필요}

### 🔴 CRITICAL ({n}건)
| 테스트 위치 | 문제 | 근거(어긋난 규약/명세) | 위임 수정 방향 |
|------|------|------|------|
| {test 경로:라인} | {구현 박제 / 현재 실패 등} | {스킬 명세·계약 대비 무엇이} | {test-writer가 어떻게 고쳐야} |

### 🟡 WARNING ({n}건)
| 테스트 위치 | 문제 | 근거 | 위임 수정 방향 |
|------|------|------|------|
| {test 경로:라인} | {구현 결합 단언 / 불변식 누락 / 경로 위반} | {규약 항목} | {완화·추가 방향} |

### 🟢 INFO ({n}건)
| 테스트 위치 | 문제 | 개선 제안 |
|------|------|------|

### ✅ 신뢰 가능 항목
- {명세/계약 근거가 분명하고 통과하는 테스트 요약}

### 요약
- CRITICAL {n} · WARNING {n} · INFO {n}
- {한 줄 총평}
- 수정이 필요하면 **test-writer**로 위임하세요. (이 에이전트는 판정·리포트만 합니다.)
- {도메인 스킬 신뢰도 의심 시: "해당 도메인 스킬은 domain-skill-reviewer 선행 점검 권고"}
```

- 이슈가 하나도 없으면 각 등급 표는 "없음"으로 표기하고 판정은 PASS로 한다.
- 실제 통과 여부((e))는 `npx vitest run` 결과를 근거로 기재하고, 추측으로 채우지 않는다.
