# test-writer 요약 반환 형식

작성·실행이 끝난 뒤 메인 에이전트에 반환할 때 아래 형식을 그대로 사용한다.
격리 컨텍스트의 탐색 기록·시행착오 로그는 포함하지 않는다.

---

```
## 🧪 테스트 작성 완료

**대상 모듈**: {src 경로}
**테스트 파일**: {test 경로 — src 미러}
**단언 소스**: {도메인 스킬 [[스킬명]] | 함수 계약(시그니처/JSDoc/불변식)}

### 실행 결과
- 명령: `npx vitest run {파일}`
- 결과: {PASS n / FAIL n} — {전체 통과 | 일부 실패}

### 커버한 케이스
- {케이스 1 — 무엇을 어떤 근거(스킬 명세/불변식)로 단언했는지}
- {케이스 2 ...}

### 주의 / 후속 (있을 때만)
- {구현 버그로 의심돼 테스트로 노출된 부분 — 구현은 고치지 않았음}
- {도메인 스킬 신뢰도 의심 → domain-skill-reviewer 선행 권고}
- {대상이 모호해 확인이 필요한 질문}
```

- 전부 통과하고 후속 사항이 없으면 "주의 / 후속" 섹션은 생략한다.
- 통과시키려 단언을 약화시킨 경우 그 사실을 반드시 "주의 / 후속"에 적는다.
