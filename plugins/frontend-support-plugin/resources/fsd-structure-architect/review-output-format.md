# fsd-structure-architect 결과 출력 형식

에이전트가 사용자에게 최종 결과를 보여줄 때 아래 형식을 그대로 사용한다.

---

## 생성(Scaffold) 모드 결과 형식

```
## 🏗️ FSD 구조 생성 완료

**대상 경로**: {루트 경로}
**생성 계층**: {app, pages, features, entities, shared ...}

### 생성된 트리
{실제 만든 디렉토리·파일 트리}

### 적용한 FSD 규칙
- 계층: {나열}
- 슬라이스: {나열}
- 세그먼트: {나열}
- Public API(index.ts): {몇 개 생성}

### 다음 단계 제안
- {비어 있는 슬라이스에 무엇을 채울지 등}
```

---

## 검토(Review) 모드 결과 형식

```
## 🔍 FSD 구조 검토 결과

**대상 경로**: {검토한 경로}
**판정**: {PASS / 개선 필요}

### 🔴 CRITICAL ({n}건)
| 위치 | 문제 | 위반 규칙 | 수정 방향 |
|------|------|----------|----------|
| {경로} | {문제 요약} | {의존성 역방향 등} | {구체적 수정안} |

### 🟡 WARNING ({n}건)
| 위치 | 문제 | 위반 규칙 | 수정 방향 |
|------|------|----------|----------|

### 🟢 INFO ({n}건)
| 위치 | 문제 | 개선 제안 |
|------|------|----------|

### ✅ 규칙 준수 항목
- {잘 지켜진 부분 요약}

### 요약
- CRITICAL {n} · WARNING {n} · INFO {n}
- {한 줄 총평}
```

- 이슈가 하나도 없으면 각 등급 표는 "없음"으로 표기하고 판정은 PASS로 한다.
- 근거 없는 추측성 이슈는 등록하지 않는다.
